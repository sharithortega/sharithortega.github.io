Diseño de software

Introducción

Diseño se define en [IEEE610.12-90] como "la definición de la arquitectura componentes interfaces y otras características de un sistema o componente" y "el resultado de ese proceso". La ingeniería de software es una actividad del ciclo de vida basada en el análisis de los requisitos del software para obtener una descripción de la estructura interna del software que es la base para su creación.

Más específicamente el diseño del software (resultado) debe describir la arquitectura del software la descomposición del software la organización de los componentes y las interfaces entre los propios componentes. También debe describir los componentes con suficiente detalle para que puedan construirse.

El diseño de software juega un papel importante en el desarrollo de software: permite a la ingeniería de software producir diferentes modelos para la solución de desarrollo. Podemos analizar y evaluar estos modelos para determinar si se pueden utilizar para satisfacer sus necesidades.

También podemos explorar y evaluar varias soluciones y compromisos alternativos. Finalmente podemos usar los modelos resultantes para planificar actividades de desarrollo posteriores además de usarlos como entrada o punto de partida para crear y probar una lista de estándares de procesos del ciclo de vida del software por ejemplo IEEE/EIA 12207 [IEEE12207.0-96.] la ingeniería de software consta de dos actividades que encajan entre el post-análisis de software y la construcción de software:

  • Diseño de arquitectura de software (a veces llamado diseño de alto nivel): descripción de la estructura y organización del software al más alto nivel e identificación de los varios componentes.
  • Diseño detallado del software: descripción suficiente de cada componente para explicar su estructura.

Aspectos básicos del diseño de software

Se deben considerar varios principios básicos al diseñar software. Hay algunas preocupaciones que todo software debe abordar como la calidad del rendimiento. Otra cuestión importante es cómo se descomponen organizan y forman los paquetes de software. Es tan importante que todos los diseños deben abordarlo de una forma u otra (consulte el Tema 1.4 Técnicas y la Sección 6 Métodos que permiten el diseño de software). Por el contrario otras versiones "abordan un aspecto específico del comportamiento del software que está fuera de alcance pero afecta alguna área de soporte". Los aspectos generalmente no son unidades de descomposición funcional de software sino funciones que afectan sistemáticamente el funcionamiento o la semántica de los componentes‖ (Kic97). Algunas de estas cuestiones multidisciplinarias clave son (en orden alfabético):

  • Concurrencia: Dividir el software en procesos tareas y subprocesos y dividirlos en versiones de eficiencia atomicidad sincronización y programación relacionadas. [Jef00:c5; 2 de mar: CDS; May97:c30; Pre04: c9]
  • Control y enrutamiento de eventos: Cómo orquestar el flujo de datos y controlar eventos reactivos y temporales utilizando diversos mecanismos como la invocación y los servicios de replicación implícita.

¿Qué es el diseño de software?

El diseño de software es el proceso de crear una representación abstracta de la arquitectura y los componentes del software. El diseño describe cómo se organizará el software y cómo interactuarán sus diferentes partes.

Modelos de diseño de software

Existen diferentes modelos de diseño de software que se utilizan para estructurar y organizar el código fuente. Algunos modelos de diseño comunes incluyen:

  • Diseño modular: Divide el software en módulos independientes y bien definidos.
  • Diseño orientado a objetos: Encapsula datos y comportamientos en objetos que interactúan entre sí.
  • Diseño basado en componentes: Utiliza componentes reutilizables para construir el software.

Actividades del Área de Diseño

El Área de Diseño en la Guía SWEBOK define las actividades que se deben llevar a cabo para diseñar la arquitectura y los componentes del software. Estas actividades incluyen:

  • Diseño arquitectónico: Define la estructura general del software incluyendo sus componentes principales y sus relaciones.
  • Diseño detallado: Describe los componentes individuales del software sus interfaces y sus implementaciones.
  • Documentación del diseño: Registra las decisiones de diseño tomadas y la arquitectura del software.

Importancia del Área de Diseño

El Área de Diseño es fundamental para asegurar que el software sea modular reutilizable mantenible y extensible. Un buen diseño de software facilita el desarrollo las pruebas y el mantenimiento del software a lo largo de su ciclo de vida.

Aspectos a tener en cuenta

Se tiene que tener en cuenta a la hora de diseñar un software una serie de principios claves. Algunas son preocupaciones que todo software debe tratar. Por ejemplo funcionamiento de la calidad como se organiza de una forma práctica y funcional que se adapten a las necesidades del usuario.
